# Snipcast

Десктоп-приложение для палитры текстовых шаблонов (macOS и Windows), собранное на **Tauri 2** и React.

## Требования

- [Rust](https://www.rust-lang.org/tools/install) (stable)
- Node.js 18+

## Запуск

```bash
cd /path/to/Snipcast
npm install
npm run tauri dev
```

Сборка установщика:

```bash
npm run tauri build
```

## Поведение (MVP)

- Иконка в **системном трее**; окно по умолчанию скрыто.
- Глобальная горячая клавиша: **⌘+Shift+Space** (macOS) / **Ctrl+Shift+Space** (Windows) — `CommandOrControl+Shift+Space`.
- Левый клик по иконке трея или пункт меню «Show palette» открывает палитру.
- **Esc** или потеря фокуса окна скрывает палитру.
- Окно **непрозрачное** с фоном `#1b1b1f` (без «ломаной» прозрачной рамки macOS); скругление у контента **внутри** окна. При открытии поле поиска **очищается**.
- После выбора шаблона текст кладётся в буфер, палитра закрывается, **возвращается фокус в предыдущее приложение** и выполняется **Cmd+V / Ctrl+V** (чтобы вставка попала туда, где был курсор).

### macOS: доступ для автоматической вставки

Нужны права у **Snipcast**:

1. **Конфиденциальность и безопасность → Универсальный доступ** — включить для Snipcast (для **enigo**: симуляция Cmd+V).
2. **Автоматизация** (или запрос при первом запуске) — разрешить Snipcast управлять **System Events** (для **osascript**: активация нужного приложения).

Без этого текст всё равно копируется в буфер, но авто-вставка в другое приложение может не сработать.

### Windows

Для **SetForegroundWindow** и эмуляции **Ctrl+V** может потребоваться запуск от имени администратора в редких политиках; в обычной среде достаточно стандартного запуска.

- На macOS для агента в трее используется `ActivationPolicy::Accessory` (без иконки в Dock в типичном режиме).
